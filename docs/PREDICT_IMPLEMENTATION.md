# Predict (Digital) Mode - 技术实现指南

## 概述
Predict模式是一个几何形状智能识别和自动补全系统，通过数学算法分析用户绘制轨迹，识别其代表的几何图形并自动生成标准化的形状。

## 核心架构

### 1. 输入流处理
- **实时坐标采集**: 持续捕捉用户绘制轨迹点流
- **预处理过滤**: 去除噪声点、异常跳动和重复数据
- **轨迹特征提取**: 提取关键几何特征（端点、拐点、曲率变化）

### 2. 特征分析算法
- **角度检测**: 使用向量夹角计算识别尖角点
- **线段拟合**: 应用最小二乘拟合检测直线段
- **曲线拟合**: 使用圆弧/椭圆拟合法匹配弯曲轨迹

### 3. 形状识别引擎
- **模板匹配**: 与预定义几何标准进行匹配
- **分类器系统**: 基于几何特征的分级分类算法
- **概率评估**: 分析各种形状的可能性评分

## 具体实现步骤

### 第一阶段：坐标数据预处理
1. **降噪采样**
   - 使用高斯加权平均减少轨迹随机噪音
   - 应用Douglas-Peucker算法进行轨迹简化
   
2. **关键点检测**
   - 计算每点的曲率变化率
   - 识别高曲率变化点（角点）
   - 保存极值点（最大/最小坐标值点）

### 第二阶段：几何特征分析
1. **角度特征计算**
   ```
   # 计算三点之间的内角
   对于轨迹中的每个非端点 p_i
     向量 v1 = p_i - p_(i-5)
     向量 v2 = p_(i+5) - p_i
     夹角 θ = acos(dot(v1, v2) / (|v1| * |v2|))
     如果 θ 非常接近 90°, 记录为可能的矩形角点
   ```

2. **轮廓逼近算法**
   - 寻找包围盒（最小/最大坐标）
   - 计算纵横比（宽高比判断）
   - 检查闭合性（首尾距离判断）

### 第三阶段：形状分类识别
1. **层次化分类器**
   - **初步筛选**:
     - 开放路径 → 线段、角度、曲线
     - 封闭路径 → 三角形、矩形、圆形等多边形
   - **细粒度分类**:
     - 特征数量 → 3点三角形、4点四边形
     - 边长关系 → 等边三角形 vs 普通三角形
     - 角度特性 → 直角 vs 锐角/钝角

2. **评分系统**
   ```
   每种候选形状获得一个评分，考虑：
   - 角点数量匹配度
   - 边长/角度一致性
   - 拟合误差（原始轨迹vs识别形状的偏差）
   - 闭合/开放性
   ```

### 第四阶段：几何校正与优化
1. **智能形状校正**
   - 矩形: 自动校正直角，优化长宽比例
   - 圆形: 自动调整为标准圆，匹配面积
   - 线段: 输出端点直接连接
   
2. **生成目标几何**
   - 基于识别结果生成标准几何对象
   - 保留原有样式（颜色、粗细）但应用理想几何特性

## 算法细节

### 角点检测算法
- 使用向量夹角（而非导数法）提升鲁棒性
- 跨距3个点避免微小抖动影响
- **自适应阈值**：开放图形使用约63°阈值，封闭图形使用约32°阈值
- **距离自适应**：基于包围盒大小的动态合并阈值（5% bboxSize）
- **闭环角点补充**：检测起点/终点的夹角，确保矩形4个角都被捕获

### 闭环检测与处理
- 首尾距离小于包围盒尺寸20%视为闭环
- 适当预留缺口避免线条自动连接
- 对封闭图形启用更敏感的角点检测

### 面积守恒优化
- 保持视觉感受的面积一致性
- 修正形状因几何校正带来的面积变化

## 已知局限性和问题

### 已修复问题 ✅

#### 1. 角点检测阈值优化 (FIXED)
- **原问题**: 固定阈值72°无法捕获矩形的90°角点
- **解决**: 实现自适应阈值 - 封闭图形使用32°(π*0.18)，开放图形使用63°(π*0.35)
- **结果**: PERFECT矩形现在能正确检测4个角点

#### 2. 矩形vs三角形分类混淆 (FIXED)
- **原问题**: 依赖内角和导致三角形/矩形误判
- **解决**: 优先使用角点数量和角度特征（直角检测）
- **结果**: 4角无直角的形状被正确识别为三角形

#### 3. 闭环起点角点丢失 (FIXED)
- **原问题**: 矩形第4个角（起点/终点）未被检测
- **解决**: 对封闭形状额外检测起点处的夹角
- **结果**: 完美矩形能正确识别4个角

### 待解决问题 ⚠️

#### 1. 三角形角点过量检测
- **问题**: 部分三角形被检测出4个角点（包含假角点）
- **影响**: `perfect_obtuse`, `handdrawn_isosceles` 等测试用例
- **原因**: 轨迹平滑处的微小弯曲被误认为角点
- **可能解决**: 增强角点验证，要求角点间形成明显边

#### 2. 手动绘制矩形识别率低
- **问题**: 用户手绘的矩形（manual_2026-03-12-A/D）识别失败
- **表现**: 被识别为 polyline 或 triangle
- **原因**: 手绘不够规整，角点不明显或边不直
- **可能解决**: 增加形状拟合容错率或提供交互式校正

#### 3. 圆弧识别问题
- **问题**: `perfect_semi_arc` 检测失败（置信度为0）
- **状态**: 按用户要求暂时忽略
- **原因**: 弧线的曲率特征与直线/曲线区分困难

#### 4. 复杂图形分割
- **问题**: 多段图形（如带缺口的形状）难以准确分割
- **表现**: 经常误识别为简单几何组合
- **原因**: 缺乏子轮廓分离算法

#### 5. 参数自适应不足
- **问题**: 无法自适应不同用户绘制风格
- **影响**: 快速绘制 vs 慢速绘制的识别率差异
- **原因**: 参数固定，未考虑绘制速度/密度
