## 三、根本原因分析

### 1. 角点检测算法问题
- **阈值过于严格**: 部分矩形角点(特别是手绘圆润的角)未达到检测阈值
- **合并逻辑问题**: 邻近角点被错误合并，导致4角矩形被识别为2-3角
- **角度计算偏差**: 对于非标准90°角(如80-100°范围)的分类不够精确

### 2. 形状分类器混淆
- **三角形 vs 矩形判定**: 当检测到3个角点时，系统倾向于判定为三角形而非"缺一角"的矩形
- **2角点处理**: 2个角点的情况缺乏明确的矩形识别逻辑，容易被归类为开放图形
- **内角和依赖**: 过度依赖内角和特征(180°=三角形, 360°=四边形)，忽略了实际几何分布

### 3. 完美几何图形识别问题
- **perfect_rectangle**: 标准矩形可能因过于"完美"而被误判为其他形状
- **perfect_equilateral**: 等边三角形的60°角可能被误识别为其他特征
- **perfect_semi_arc**: 半圆弧形的检测逻辑存在根本缺陷
