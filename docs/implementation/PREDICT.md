# Predict (Digital) Mode - 技术实现指南

## 概述
Predict模式是一个几何形状智能识别和自动补全系统，通过数学算法分析用户绘制轨迹，识别其代表的几何图形并自动生成标准化的形状。

## 核心架构

### 1. 输入流处理
- **实时坐标采集**: 持续捕捉用户绘制轨迹点流
- **预处理过滤**: 去除噪声点、异常跳动和重复数据
- **轨迹特征提取**: 提取关键几何特征（端点、拐点、曲率变化）

### 2. 特征分析算法
- **角度检测**: 使用向量夹角计算识别尖角点
- **线段拟合**: 应用最小二乘拟合检测直线段
- **曲线拟合**: 使用圆弧/椭圆拟合法匹配弯曲轨迹

### 3. 形状识别引擎
- **模板匹配**: 与预定义几何标准进行匹配
- **分类器系统**: 基于几何特征的分级分类算法
- **概率评估**: 分析各种形状的可能性评分

## 识别流水线 (Classification Pipeline)

### 阶段1: 形状特征分析 (`analyzeShapeFeatures`)
```
输入: Point[] (原始轨迹点)
输出: ShapeFeatures (完整特征分析)

步骤:
1. 计算转角 (computeAngles) - 使用向量余弦相似度
2. 检测闭环 (detectClosedShape) - 首尾距离 + 相对包围盒检查
3. 角点检测 (findCorners) - 自适应阈值 + 距离合并
4. 曲率计算 (computeCurvatures) - 局部曲率分析
5. 起点角点补充 - 封闭图形额外检测起点夹角
6. 角点角度特征 (computeCornerAngleFeatures) - 内角分析
```

### 阶段2: 形状分类评分 (`classifyShapeImproved`)
```
输入: Point[], ShapeFeatures
输出: ImprovedClassification (分类结果 + 所有评分)

评分系统 (按优先级排序):
1. line      - 低曲率 + 少角点 + 开放
2. angle     - 单个尖角 + 开放
3. polygon   - 4+角点 (通用四边形)
4. triangle  - 3角点 + 角度合理 + 封闭
5. rectangle - 4角点 + 3+直角
6. square    - 4角点 + 3+直角 + 宽高比≈1
7. circle    - 0-3角点(假角点) + 均匀曲率 + 封闭
8. ellipse   - 0-3角点 + 基本均匀曲率 + 明显长短轴
9. arc       - 部分圆弧特征
10. curve    - 不规则曲线
11. polyline - 多段线 + 开放
```

**重要评分规则:**

**Triangle 评分 (3角点验证):**
```
验证条件 (必须同时满足):
1. 最小角度 ≥ 25° (排除假角点)
2. 最大角度 ≤ 160° (排除异常大角)
3. 小角度数量 ≤ 1 (<45°的角不超过1个)

失败惩罚:
- 角度异常 → 分数降低至 0.25
- isValidTriangleCorners = false
- 不再获得"角度符合三角形特征"奖励
```

**Circle 评分 (3角点处理):**
```
不再直接拒绝3+角点，而是:
1. 检查是否为有效三角形 (同上验证)
2. 有效三角形 → 拒绝 (多边形特征明显)
3. 无效三角形 (假角点) → 继续评分，但基础分仅 0.05

评分要素:
- 0角点: +0.3
- 1-2角点: +0.2  
- 3假角点: +0.05
- 曲率均匀: +0.1~0.3
- 周长匹配: +0.15~0.25
- 接近正圆: +0.08~0.15
```

**Rectangle 评分 (3角点降级):**
```
3角点情况 (可能是矩形缺一角):
- 2+直角: score += 0.25 (原0.45)
- 1直角: score += 0.15 (原0.30)
- 无直角: score += 0.10

直角奖励区分:
- 4角点+2+直角: +0.2 (强矩形)
- 3角点+2+直角: +0.1 (弱矩形，可能是三角形)
```

### 阶段3: 评分排序与后处理
```
排序规则:
1. 主要排序: 分数降序
2. 平局打破 (分数差 < 0.15):
   - triangle vs rectangle: 看角点数量和直角数
   - 4角+2+直角 → rectangle
   - 4角+无直角 → triangle (多余角点)
   - 3角 → triangle

后处理规则:
1. 4角+无直角+rectangle最佳 → 强制转为 triangle
2. 3角+2+直角+isRectLike → 保持 rectangle
3. 4+角点多边形优先级:
   - 3+直角 + rectScore≥0.7 → 保持 rectangle/square
   - 否则 → polygon (通用四边形)
```

### 阶段4: 形状点生成 (`generateShapePointsImproved`)
```
根据分类类型生成标准几何点:
- line: [起点, 终点]
- angle: [起点, 角点, 终点]
- triangle: [角点1, 角点2, 角点3, 角点1]
- rectangle/square: [角点1, 角点2, 角点3, 角点4, 角点1]
- polygon: [角点1, ..., 角点N, 角点1]
- circle: 32点均匀采样圆
- ellipse: 32点均匀采样椭圆
```

## 关键算法细节

### 角点检测算法 (findCorners)
```
参数:
- 步长: 3个点
- 封闭图形阈值: π*0.18 (~32°) - 更敏感
- 开放图形阈值: π*0.35 (~63°) - 标准
- 最小距离: max(10px, bboxSize * 0.025) - 防止角点合并过强

特殊处理:
- 2角点封闭形状 → 强制添加起点作为第3角点 (三角形补偿)
- 起点角点检测: 计算终点→起点→第3点的夹角
```

### 闭环检测 (detectClosedShape)
```
判断条件 (满足任一即视为闭环):
1. 首尾距离 < bboxSize * 0.25
2. 首尾距离 < pathLength * 0.08
3. 首尾距离 / bboxSize < 0.10 (相对检查)
```

### 矩形评分关键逻辑
```
4角点评分:
- 3+直角 (90°±30°): score += 0.7 (强矩形)
- 2直角: score += 0.5 (中等, 可能是梯形)
- <2直角: score += 0.35 (弱)

附加分:
- 2+直角: +0.2
- 低曲率: +0.05~0.1
- 合理宽高比: +0.05
- 良好封闭: +0.05
```

### 多边形优先级规则
```
何时polygon胜过rectangle:
1. 无3+直角 或 rectScore < 0.7
2. polygonScore > rectScore * 1.1 (显著更高)

何时rectangle胜过polygon:
1. 3+直角 + rectScore ≥ 0.7
2. 明显矩形特征 (isRectLike)
```

## 圆与椭圆的关系设计

### 当前实现: 独立分类
```
circle 和 ellipse 是两个独立的形状类型:
- circle: 要求曲率完全均匀, 无明显长短轴
- ellipse: 允许轻微不均匀, 有明显长短轴差异

区别逻辑:
- circle评分高 → 识别为 circle
- circle评分低 + ellipse评分高 → 识别为 ellipse
- 用户意图: 圆是圆, 椭圆是椭圆 (不是特例关系)
```

### 设计理由
**方案A: 圆是椭圆的特例** (未采用)
- 优点: 数学上正确, 统一处理
- 缺点: 用户画圆时可能被识别为"扁椭圆", 体验差

**方案B: 独立分类** (当前采用)
- 优点: 
  - 用户画圆期望得到正圆, 不是椭圆
  - 可以分别优化识别逻辑
  - 正方形vs矩形也采用类似设计
- 缺点: 需要维护两套评分逻辑

### 实际使用建议
- 如果用户绘制接近正圆 → 纠正为 perfect circle
- 如果用户绘制明显椭圆 → 纠正为 perfect ellipse
- 保持两者独立更符合用户直觉
